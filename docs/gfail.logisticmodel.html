gfail.logisticmodel¶

This module contains functions and class definitions for running forward models of models based on logistic regression.

class gfail.logisticmodel.LogisticModel(shakefile, config, uncertfile=None, saveinputs=False, slopefile=None, bounds=None, numstd=1, slopemod=None)[source]¶

Bases: object

Set up the logistic model :param shakefile: Path to shakemap grid.xml file for the event. :type shakefile: str :param config: configobj object defining the model and its inputs. Only

one model should be described in each config file.
Parameters:
  • uncertfile (str) – Path to uncertainty.xml file.
  • saveinputs (bool) – Save input layers as Grid2D objects in addition to the model? If false (the default), it will just output the model.
  • slopefile (str) – Optional path to slopefile that will be resampled to the other input files for applying thresholds. OVERWRITES VALUE IN CONFIG.
  • bounds (dict) –

    Default of None uses ShakeMap boundaries, otherwise a dictionary of boundaries to cut to like

    bounds = {
    'xmin': lonmin, 'xmax': lonmax,
    'ymin': latmin, 'ymax': latmax
}
  • numstd (float) – Number of +/- standard deviations to use if uncertainty is computed.
  • slopemod (str) – How slope input should be modified to be in degrees: e.g., np.arctan(slope) * 180. / np.pi or slope/100. (note that this may be in the config file already).
getEquations()[source]¶

Method for LogisticModel class to extract strings defining the equations for the model for median ground motions and +/- one standard deviation (3 total). :returns:

Three equations: equation, equationmin, equationmax, where
  • equation is the equation for median ground motions,
  • equationmin is the equation for the same model but with median ground motions minus 1 standard deviation, and
  • equationmax is the same but for plus 1 standard deviation.
